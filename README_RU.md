# Асинхронный Telegram-бот обработки медиа и поиска музыки

Высокопроизводительный асинхронный Telegram-бот, разработанный для поиска музыки, доставки метаданных, динамической генерации обложек и наложения аудиоэффектов (ремиксов) на лету. Бот интегрирует несколько источников метаданных (Deezer, YouTube Music, Spotify), поддерживает парсинг текстов песен и предоставляет полностью локализованный интерфейс на 6 языках.

Разработано на **Python 3.12**, с использованием **aiogram v3**, **Celery**, **PostgreSQL** и **Redis**.



## 🚀 Основные возможности

*   **🔍 Поиск по нескольким источникам**: Одновременный опрос API Deezer, YouTube Music и Spotify для получения метаданных треков, обложек альбомов и 30-секундных превью.
*   **🌐 Локализация на 6 языков**: Полностью переведенный интерфейс, поддерживающий **немецкий, английский, испанский, казахский, русский и украинский** языки. Выбор языка регулируется через кастомное middleware для aiogram, меняющее локаль «на лету» на основе настроек пользователя.
*   **🎛️ Асинхронный движок аудиоэффектов**: Тяжелые операции обработки аудио (изменение скорости, сдвиг тональности, реверберация и конвертация форматов) вынесены в фоновые задачи **Celery** с использованием утилиты **FFmpeg**, что исключает блокировку команд бота.
*   **🎨 Эффект стеклянной обложки (Glassmorphism)**: Автоматическая генерация красивых карточек трека с наложением обложки альбома на размытый полупрозрачный фон с помощью библиотеки **Pillow (PIL)**.
*   **⚡ Умное кэширование и БД**:
    *   **PostgreSQL** (`asyncpg` + SQLAlchemy) хранит информацию о пользователях, лайках и кэширует `file_id` Telegram, чтобы избежать повторной загрузки одинаковых аудиофайлов.
    *   **Redis** выступает в роли кэша поисковых запросов, брокера сообщений Celery и трекера для защиты от спама (anti-flood).

## 🛠️ Технологический стек

*   **Язык**: Python 3.12 (асинхронный стек async/await)
*   **Фреймворк**: aiogram v3 (роутинг, FSM, кастомные middleware)
*   **База данных и кэш**: PostgreSQL, Redis, SQLAlchemy 2.0, asyncpg
*   **Очередь задач**: Celery (фоновая обработка аудио)
*   **Обработка медиа**: FFmpeg, Pillow (PIL), libsndfile
*   **Деплой**: Docker, Docker Compose, Nginx (для мониторинга healthcheck)

## ⚙️ Быстрый старт

> **Правовая оговорка:** По умолчанию бот доставляет только публичные метаданные и 30-секундные превью. Полноценное прослушивание треков настраивается через подключение `LicensedAudioSource`, указывающий на ваш собственный лицензионный каталог.

### 1. Настройка переменных окружения
Скопируйте шаблон конфигурации в `.env`:

```bash
cp .env.example .env
```

Отредактируйте `.env`, указав как минимум:
*   `BOT_TOKEN` — Токен бота от [@BotFather](https://t.me/BotFather)
*   `BOT_USERNAME` — Юзернейм вашего бота
*   `GENIUS_TOKEN` — (Опционально) Токен с genius.com для получения текстов песен

### 2. Запуск через Docker (Рекомендуется)
Эта команда соберет и запустит все необходимые сервисы (Бот, Postgres, Redis, Celery воркер и панель мониторинга Flower):

```bash
docker compose up -d --build
```

Просмотр логов:
```bash
docker compose logs -f bot
```

*   **Проверка работоспособности**: `http://localhost:8080/health`
*   **Мониторинг задач Celery (Flower)**: `http://localhost:5555`

### 3. Локальный запуск (без Docker)
Если вы хотите запустить проект непосредственно в системе:

1.  Запустите инфраструктуру БД и кэша:
    ```bash
    docker compose up -d postgres redis
    ```
2.  Установите системные утилиты (FFmpeg обязателен для обработки аудио):
    ```bash
    sudo apt install ffmpeg libsndfile1
    ```
3.  Создайте виртуальное окружение Python и установите зависимости:
    ```bash
    python3 -m venv .venv
    source .venv/bin/activate
    pip install -r requirements.txt
    ```
4.  Запустите бота:
    ```bash
    python -m bot.main
    ```
5.  В отдельном терминале запустите Celery воркер для аудиоэффектов:
    ```bash
    celery -A workers.fx_worker:celery_app worker --loglevel=info --concurrency=2
    ```
